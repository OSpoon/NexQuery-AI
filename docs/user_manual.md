# NexQuery AI 用户手册

欢迎使用 NexQuery AI。

NexQuery AI 旨在帮助**运营人员与非技术用户**轻松、安全地获取数据。我们的目标是让您在**最大程度减少对技术部门的日常依赖**的情况下，即可通过简单的自然语言对话，自主完成数据查询与导出工作。

**协作模式**:
*   **专业人士 (DBA/技术支持)**: 负责配置复杂的数据源连接、预置高频查询任务和配置推送渠道。
*   **运营人员**: 负责“一键执行”任务，或直接查收邮件/群消息中的数据报表。

本文档将指导您如何使用平台的各项功能。如需了解底层逻辑代码，请查阅 [开发指南](./development.md)。

## 1. 登录与安全

*   **会话有效期**: 默认 Web 会话有效期为 **24小时**。超时需重新登录。

*   绑定后，登录时需输入 6 位动态验证码。

## 2. 工作台 (Dashboard)
登录后默认进入工作台，这是您的全景视图，提供实时的系统健康监测与业务活动追踪。

### 2.1 运行环境健康度 (Environment Health)
系统会实时检测所有关键组件的连通性。如果出现红色的 `ERR` 状态，系统会直接显示错误详情以便排查。
*   **System Database**: 监控 NexQuery 系统自身的数据库连接状态。
*   **Data Sources Integrity**:
    *   自动扫描所有状态为 `Active` 的外部数据源。
    *   对每个数据源执行连通性测试 (Ping/Select 1)。
    *   **故障诊断**: 若某个数据源连接失败（如密码过期或网络不通），面板会展开显示具体失败的数据源名称及错误原因。

### 2.2 核心指标 (Key Metrics)
*   **Data Sources**: 当前已激活且连通的数据源总数。
*   **SQL Tasks**: 已配置并保存的固定报表任务数。
*   **Executions**: 历史累计执行查询的总次数（包含通过 AI、编辑器及定时任务触发的所有查询）。
*   **Overall Health**: 根据过去 7 天的查询成功率计算的系统健康指数。

### 2.3 趋势与排行 (Trends & Rankings)
*   **Activity Trend**: 过去 7 天的查询量趋势，区分成功与失败状态。
*   **Top Sources**: 访问频率最高的热门数据源排行，帮助管理员识别热点业务库。
*   **Top Users**: 系统活跃用户排行。
*   **Recent Live**: 实时滚动的最新查询日志，点击日志状态可查看详细的执行信息。

*   **Recent Live**: 实时滚动的最新查询日志，点击日志状态可查看详细的执行信息。

## 3. 数据源管理 (前置步骤)

*注意：此章节通常由**技术人员**操作配置。*

在进行查询之前，您必须先配置数据源。请进入 **Data Sources** 菜单。

### 3.1 添加数据源
支持以下类型：
*   **MySQL**: 标准 MySQL 数据库。
*   **PostgreSQL**: 标准 PostgreSQL 数据库。
*   **API**: 用于查询 HTTP API 接口（通过 curl 命令）。

### 3.2 高级配置 (Advanced Config)
点击列表中的 "Settings" 图标可配置数据处理规则。这些配置将直接影响查询结果的展示。

**自动同步 (Auto-Sync)**: 配置成功保存后，系统会自动启动后台任务同步表结构元数据至向量数据库，确保 AI 智能助手能立即识别新添加的数据源。

**自动同步 (Auto-Sync)**: 配置成功保存后，系统会自动启动后台任务同步表结构元数据至向量数据库，确保 AI 智能助手能立即识别新添加的数据源。

*   **配置项**:
*   **数据脱敏 (Data Masking)**: 保护敏感信息。
*   **Mobile**: 手机号脱敏 (e.g. `138****1234`)。
*   **Email**: 邮箱脱敏。
*   **ID Card**: 身份证脱敏。
*   **Password**: 完全隐藏。
*   **枚举映射 (Enum Mapping)**: 将数据库存储的代码 (e.g. `1`, `0`) 自动转换为可读文本 (e.g. `启用`, `禁用`)。
*   **别名 (Alias)**: 为晦涩的字段名设置友好的显示名称。

### 3.3 PII 自动发现 (Auto-Discovery)
当您添加或更新数据源并开启 **自动同步 (Auto-Sync)** 后，系统会自动触发 AI 敏感数据扫描：
1.  **AI 识别**: 系统调用大模型分析表结构与字段注释，智能识别手机号、邮箱、身份证、银行卡等敏感信息。
2.  **自动配置**: 识别到的字段会自动应用相应的**数据脱敏 (Data Masking)** 规则，并在高级配置中标记为 "AI Discovered"。
3.  **通知**: 扫描完成后，您会在右上角收到通知。您可以在高级配置中人工复核并调整这些规则。

## 4. SQL 查询与执行 (核心功能)

配置好数据源后，进入 **Query Execution** 页面进行数据检索。

### 4.1 选择数据源
*   **第一步**: 在页面左上角的下拉框中，选择您在 [第 3 章](#3-数据源管理) 中配置的数据源。
*   **注意**: 只有选择了数据源，SQL 编辑器和表结构树才会激活。

### 4.2 SQL 编辑器
*   支持语法高亮与智能提示。
*   **Intellisense**: 输入表名或字段名时会自动补全。
*   **格式化**: 点击 "Format" 按钮一键美化 SQL 代码。
*   **执行**: 点击 "Run" (或 `Cmd/Ctrl + Enter`) 执行查询。

### 4.3 执行结果
*   结果以表格形式展示，支持排序和分页。
*   **应用脱敏**: 如果配置了规则，敏感数据在此处会显示为 `****`。
*   **安全提示**: 若查询执行失败，为防止数据库结构泄露，系统仅会提示通用的“数据库操作异常”，详细错误请联系管理员查看服务端日志。
*   **导出**: 点击 "Export" 支持导出为 CSV 或 JSON 文件。
*   **导出**: 点击 "Export" 支持导出为 CSV 或 JSON 文件。

## 5. AI 智能助手

点击页面右下角的 **Brain (大脑)** 图标或使用编辑器上方的 AI 功能。

### 5.1 Text-to-SQL (生成)
1.  打开 AI 对话框。
2.  用自然语言描述您的需求，例如：“查询最近一周注册活跃用户”。
3.  AI 生成 SQL 后，会自动**填充到 SQL 编辑器**中。
4.  您可以直接点击 "Run" 执行，或手动微调 SQL。

### 5.2 智能优化 (Optimize)
*   点击 **"Optimize SQL"**，AI 将分析当前编辑器中的 SQL 代码。
*   提供索引创建建议和查询重写方案，帮助您提升查询效率。

### 5.3 AI 反馈与自动学习 (Feedback & Learning)
*   **按赞即学习**: 当您点击回复气泡下方的“👍 (Helpful)”按钮时，系统会自动将该问答对提取并加入 AI 知识库。
*   **修正建议**: 如果生成的 SQL 有误，您可以点击“👎”并提供修正后的 SQL。管理员审核后，这些知识将显著提升 AI 下次的表现。

### 5.3 可观测性 (Mind Chain)
为了透明化 AI 的决策过程，您可以查看思维链：
*   **Reasoning**: 看到 AI 是如何一步步思考的。
*   **Tool Usage**: 查看 AI 调用了哪些工具。
*   **Retry Loop**: 如果 AI 第一次生成的 SQL 有误，您可以看到它如何根据错误信息进行自我修正。

### 5.4 安全护栏
*   系统会拦截 `DROP` 和 `TRUNCATE` 等高危破坏性命令。
*   所有生成的 SQL 均经过基础语法验证。
*   所有生成的 SQL 均经过基础语法验证。

*   **作用**: 您的反馈将进入管理后台，帮助管理员优化系统准确性。

### 5.6 主动消解歧义 (Disambiguation)
*   **交互机制**: 当您的提问过于模糊（例如：“查一下活跃用户”），AI 可能不确定对应的业务定义。
*   **操作方式**: AI 会停止生成并弹出一组选项。您只需点击符合预期的选项，AI 即可继续为您生成准确的 SQL。

## 6. 任务调度与推送

在 **Query Tasks** 中，我们可以实现“一次配置，自动运行”。

> **典型场景**: 技术人员配置任务 -> 运营人员定时收邮件/消息。

### 6.1 任务配置 (Config)
*   **编写 SQL**: 输入经过验证的业务 SQL。
*   **调度规则**:
    *   **Recurring**: 使用 Cron 表达式设置周期 (e.g. `0 9 * * 1` 每周一早9点)。
    *   **One-off**: 指定具体时间执行一次。

### 6.2 结果推送 (Delivery)
系统支持将执行结果自动推送到您的工作台：

*   **Email 通知**:
    *   在任务配置的 `Recipients` 字段输入接收人邮箱。
    *   系统会自动发送包含数据附件 (CSV) 的邮件。
*   **IM 群机器人 (Webhook)**:
    *   在 `Webhook URL` 字段填入企业微信、钉钉或飞书机器人的地址。
    *   任务完成后，数据摘要将直接推送到工作群。

### 6.3 结果查看
*   **History**: 即使通过邮件发送，系统也会保留一份执行快照在 **Query History** 中。
*   **History**: 即使通过邮件发送，系统也会保留一份执行快照在 **Query History** 中。

## 8. 系统管理

如果您是管理员，可以访问 **System Settings**。
*   **User Management**: 创建新用户、禁用违规用户或重置用户密码。
*   **Role Management**: 定义角色 (Role) 并分配菜单权限。
*   **Menu Management**: 动态配置系统菜单结构。
### 8.2 AI 配置 (AI Configuration)
在 **Settings** 页面，您可以调整 AI 引擎的全局参数：
*   **模型选择**: 切换对话模型 (Chat Model) 和嵌入模型 (Embedding Model)。
*   **超时控制 (Timeout)**:
    *   默认值为 **600秒 (10分钟)**。
    *   如果您的数据库 Schema 非常庞大，或者查询极其复杂导致 AI 经常通过超时错误，可以适当调大此数值。

## 9. 微信小程序 (Mobile)

NexQuery AI 提供了配套的微信小程序，方便您随时随地查看数据。

### 9.1 登录与绑定
1.  **一键登录**: 打开小程序，点击“微信一键登录”。
2.  **账号绑定**:
    *   首次登录时，系统会自动检测您的微信号是否已绑定 NexQuery 账号。
    *   如未绑定，系统会引导您输入 Web 端的账号密码进行绑定。
    *   绑定成功后，后续即可无需密码直接登录。

### 9.2 任务查看 (Tasks)
*   **列表**: 展示所有有权访问的查询任务。
*   **标签筛选**: 顶部提供横向滚动的标签栏，点击即可按业务标签快速筛选任务。
*   **搜索**: 顶部提供了吸顶搜索框，支持快速搜索任务名称。
*   **执行**: 点击任务进入执行页，确认参数后即可运行。

### 9.3 历史记录 (History)
*   **时间轴**: 按时间倒序展示最近的查询记录，卡片右上角显示**执行人**信息。
*   **多维筛选**:
    *   **状态**: 筛选“成功”或“失败”的记录。
    *   **标签**: 支持按任务标签聚合筛选历史记录。
    *   **搜索**: 可通过任务名称快速检索。
*   **详情页**:
    *   支持查看详细的 SQL、参数、执行时间和错误信息。
    *   **API 增强**: 对于加密的 API 任务结果，系统会**自动解密**并提供 JSON 概览。
*   **结果预览**: 对于成功的查询，支持一键进入详细结果视图（支持表格与 JSON 视图切换）。

### 9.4 个人中心 (Profile)
*   **头像同步**: 点击头像可调用微信头像并同步更新到 Web 端。
*   **账号安全**: 支持在移动端直接修改登录密码。
*   **关于我们**: 查看版本信息、用户协议及隐私政策。

## 10. AI 治理与知识库

这是系统实现“越用越聪明”的核心逻辑。

### 10.1 AI 反馈管理 (Feedback Management)
管理员可以在 **AI Feedback** 菜单查看所有用户评价：
*   **对话上下文**: 点击详情可以查看完整的对话历史和 AI 的推理过程，方便审计错误原因。
*   **追溯性**: 即使对话已被删除，反馈记录依然会保留关键的 SQL 快照。

### 10.2 知识库提炼 (Knowledge Base Promotion)
对于用户提交的修正内容，可以进行“提炼”：
1.  在反馈列表点击 **Promote**。
2.  调整关键词与标准 SQL 示例。
3.  点击提交后，该条目会进入 **Knowledge Base** 列表。

*   **发布**: 管理员审核通过后（状态改为 `Approved`），RAG 引擎才会正式在对话中引用该知识。
*   **安全性**: 未经审核 (Pending) 或已拒绝 (Rejected) 的反馈修正内容**不会**被 AI 学习，确保了 AI 思维过程的纯净度。
*   **持续优化**: 此时 AI 在遇到类似问题时，会优先参考已审核的知识库，从而实现自进化。
